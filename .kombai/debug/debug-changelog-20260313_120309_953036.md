# Debug Changelog — 20260313_120309_953036

**Task:** Enhance the Sonnenaktivität card (Solar Activity) with a live 3-layer narrative UI:
1. Strahlungsausbruch (FLR flare data)
2. Plasmawolke im Anflug (CME + speed + earth-targeting)
3. Polarlicht-Chance (CME arrival time + countdown)

Including satellite/instrument attribution on each row.

## Changes Made

### 1. Added CME types to `src/services/nasa.ts`
- **File:** `src/services/nasa.ts`
- **Change:** Added three new exported interfaces: `CmeEnlil`, `CmeAnalysis`, `CME`
- **Reason:** CME data (/DONKI/CME) was not typed before. Required to display plasma cloud speed, earth-targeting, and arrival time.
- **Revert:** Remove `CmeEnlil`, `CmeAnalysis`, `CME` interface declarations.

### 2. Added `instruments` to `SolarFlare` interface — `src/services/nasa.ts`
- **File:** `src/services/nasa.ts`
- **Change:** Added optional `instruments?: Array<{ displayName: string }>` to `SolarFlare`
- **Reason:** To show the satellite that detected the flare (e.g. "GOES-P"). Already returned by the API but was not typed.
- **Revert:** Remove `instruments` field from `SolarFlare`.

### 3. Added `cmes: CME[]` to `SpaceWeatherData` — `src/services/nasa.ts`
- **File:** `src/services/nasa.ts`
- **Change:** Added `cmes: CME[]` to `SpaceWeatherData` interface.
- **Reason:** Required to pass CME data through to the SpaceWeather component.
- **Revert:** Remove `cmes` field from `SpaceWeatherData`.

### 4. Bumped weather cache key `"weather"` → `"weather_v2"` — `src/services/nasa.ts`
- **File:** `src/services/nasa.ts`
- **Change:** Added `const WEATHER_CACHE_KEY = "weather_v2"` and used it in `getCached`/`setCache` calls.
- **Reason:** Old cached data lacks the `cmes` field. Bumping the key forces a fresh fetch for all users without manual localStorage.clear().
- **Revert:** Replace `WEATHER_CACHE_KEY` references with literal `"weather"`.

### 5. Added CME fetch to `fetchSpaceWeather` — `src/services/nasa.ts`
- **File:** `src/services/nasa.ts`
- **Change:** Added third parallel fetch to `/DONKI/CME` in `Promise.all`. Parsed result into `cmes` array. Included `cmes` in cached `SpaceWeatherData` object.
- **Revert:** Remove `cmesRes` from `Promise.all`, remove `cmes` parsing, remove `cmes` from data object.

### 6. Added solar event helper functions — `src/components/SpaceWeather.tsx`
- **File:** `src/components/SpaceWeather.tsx`
- **Change:** Added 7 pure helper functions:
  - `classifyFlareGrade(classType)` — maps X/M/C/B to color, grade label, and user-facing impact text
  - `timeAgo(dateStr, lang)` — "vor 2 Std." / "2h ago"
  - `countdown(futureDate, lang)` — "In 32 Stunden" / "In 1d 8h"
  - `classifySpeed(speed)` — "Sehr schnell" / "Very fast"
  - `getInstrumentName(instruments)` — extracts "GOES-P" from "GOES-P: GOES-P 0.5-4.0"
  - `findBestCme(cmes)` — picks most relevant CME (prefers Earth-targeted, then most recent)
  - `formatArrivalDate(dateStr, lang)` — "Sa., 15. Mär., 22:30"
- **Reason:** Clean separation of data transformation logic from JSX.
- **Revert:** Remove all 7 helper functions.

### 7. Added `BestCmeResult` interface — `src/components/SpaceWeather.tsx`
- **File:** `src/components/SpaceWeather.tsx`
- **Change:** Added `interface BestCmeResult { cme, analysis, isEarthTargeted, arrivalTime }`

### 8. Redesigned Solar Activity card — `src/components/SpaceWeather.tsx`
- **File:** `src/components/SpaceWeather.tsx`
- **Change:** Replaced old solar card (status dot + level label + "Letzter Flare" text) with a narrative event feed:
  - **Quiet state** (no flares, no CME): single green "Sonne ruhig" row
  - **Row 1 — Strahlungsausbruch**: classType badge, grade label (Extrem/Mittel/Schwach), user impact text, time-ago, satellite name ("Beobachtet: GOES-P")
  - **Row 2 — Plasmawolke im Anflug**: speed in km/s, speed classification label, earth-targeting status (green if Earth-directed), satellite name ("Bestätigt: SOHO")
  - **Row 3 — Polarlicht-Chance**: estimated arrival datetime, countdown, accuracy flag ("Genaue Vorhersage")
  - Each row uses the same `border-l-2` + translucent background pattern as the Kp tier scale.
  - Row colors: Flare = grade color (red/orange/yellow/green), CME = indigo (`#818cf8`), Arrival = gold (`#D4AF37`)
