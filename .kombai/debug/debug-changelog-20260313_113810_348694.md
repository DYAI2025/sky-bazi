## Changes Made

### 1. Fixed Tailwind v4 CSS cascade layers — `src/index.css`
- **File:** `src/index.css`
- **Change:** Moved `body` and `h1,h2,h3` base styles into `@layer base {}`. Moved all `.sky-card`, `.sky-grain`, `.status-pulse`, `.hero-gradient`, `.sky-divider`, `.funnel-glow`, `.sky-skeleton` into `@layer components {}`. Moved `@keyframes` outside layers (they are globally scoped and safe without layers).
- **Reason:** In Tailwind v4, unlayered CSS always wins over Tailwind's layered utilities. Custom classes were unlayered, which could silently break utility overrides.
- **Revert:** Remove `@layer base {}` and `@layer components {}` wrappers, restoring the original flat structure.

### 2. Upgraded sub-11px font sizes — all components + pages
- **Files:** `src/components/Header.tsx`, `src/components/ApodHero.tsx`, `src/components/FunnelCta.tsx`, `src/components/SpaceWeather.tsx`, `src/components/PlanetPositions.tsx`, `src/components/ArticleTeaser.tsx`, `src/components/Footer.tsx`, `src/pages/ArticlePage.tsx`, `src/pages/ArticlesPage.tsx`
- **Change:**
  - `text-[8px]` → `text-[10px]` (article card overlay tags — pill badges)
  - `text-[9px]` → `text-[11px]` (all kicker labels, nav items, lang toggle, section subtitles)
  - `text-[10px]` → `text-[11px]` (footer text, reading times, Kp-index labels, copyright, meta)
  - `text-[11px]` → `text-xs` (12px) (FunnelCta CTA button text)
- **Reason:** Text below 11px is unreadable in most screen conditions and fails accessibility contrast requirements. Minimum for UI labels is 11px; minimum for body/interactive text is 12px.
- **Revert:** Restore original size values per component.

### 3. Improved low-opacity text contrast — all components
- **Files:** same as above
- **Change:** Raised very low opacity text values: `0.22 → 0.35`, `0.25 → 0.35`, `0.30 → 0.40`, `0.35 → 0.40`, `0.45 → 0.50` where appropriate for muted secondary text.
- **Reason:** Text at 0.22–0.30 opacity on a dark background fails WCAG AA contrast ratio (4.5:1 for small text). Raising to ≥0.35 improves readability while preserving the subtle hierarchy.
- **Revert:** Restore original opacity values.

### 4. Fixed HTML nesting bug in ArticlePage — `src/pages/ArticlePage.tsx`
- **File:** `src/pages/ArticlePage.tsx`
- **Change:** Added a custom `p` component override in the `ReactMarkdown` `components` prop. When ReactMarkdown detects that a `<p>` node wraps only a single `<img>` element (image-only paragraph from markdown syntax `![alt](src)`), it renders the children without the `<p>` wrapper (returns `<>{children}</>`). Otherwise renders normally. Also imported the `Element` type from `hast` for type safety.
- **Reason:** ReactMarkdown wraps `![alt](src)` in a `<p>` tag. Our custom `img` renderer returns a `<figure><img/><figcaption/></figure>`. This puts a block element (`<figure>`) inside an inline element (`<p>`), which is invalid HTML per spec. React 19 logs console errors for this and it causes hydration issues.
- **Revert:** Remove the `p` component override and the `import type { Element } from "hast"` import.
